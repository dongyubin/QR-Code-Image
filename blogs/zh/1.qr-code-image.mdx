## 什么是 QR Code Image？

QR code image 通常指被保存、下载或分享为图片文件的二维码，常见格式包括 PNG、JPG 和 SVG。用户搜索这个关键词时，通常不是为了了解二维码原理，而是想马上完成一个具体任务：生成二维码图片、扫描已有图片里的二维码，或把图片链接做成二维码。

所以，一个真正好用的二维码图片工具，不能只生成黑白方块。它还需要支持截图扫描、不同二维码类型、样式设置、图片下载，以及发布前的可扫描测试。

## 用户搜索 "QR Code Image" 时真正想做什么

常见需求主要集中在这些场景：

- 为网站、Wi-Fi、社交主页、文本、PDF 链接或图片链接生成二维码图片。
- 下载 PNG、JPG 或 SVG，用在海报、菜单、标签、PPT、邮件或产品包装里。
- 从截图、照片、聊天图片、PDF 截图中扫描二维码。
- 让二维码图片比普通黑白样式更好看。
- 检查带样式的二维码是否还能被扫描器识别。

因此，最佳流程应该是浏览器端完成：输入内容、实时预览、选择样式、测试可扫性，然后再下载最终图片。

## 二维码图片格式怎么选？

### PNG

PNG 是最通用的默认选择。它边缘清晰，适合网页、文档、社交图片、邮件和日常下载。如果你只是想快速获得一个稳定的二维码图片，优先选择 PNG。

### SVG

SVG 是矢量格式，适合打印和设计软件。它可以无损放大，不会因为尺寸变大而模糊。海报、包装、标牌、贴纸、物料设计等场景更适合用 SVG。

### JPG

JPG 兼容性不错，但压缩可能破坏二维码的小模块。只有在平台不支持 PNG 或 SVG 时才建议使用 JPG，并且下载后一定要再次扫描测试。

## QR Code Image 可以包含哪些内容？

二维码图片可以编码很多类型：

- URL：网站、落地页、商品页、预约页或活动链接。
- Text：普通文本、说明、短码或离线信息。
- Wi-Fi：让访客扫码连接网络，减少手动输入密码。
- Email：打开预填好的邮件地址、标题和内容。
- Phone：用于名片、海报或线下物料的一键拨号。
- SMS：打开短信发送界面。
- vCard：保存联系人信息。
- PDF：跳转到菜单、手册、报告、目录或宣传册。
- Image URL：打开托管在网站、CDN 或对象存储上的图片。
- X Profile/Post/Share：跳转到 X 主页、具体帖子或分享入口。

需要注意的是，图片二维码通常不是把原始图片文件塞进二维码里，而是把图片的访问 URL 编码进去。先把图片上传到稳定位置，再生成对应链接的二维码。

## 如何让二维码图片更容易被扫描？

二维码可以美化，但可扫描性必须优先。建议保留这些基础条件：

- 模块颜色和背景颜色要有足够对比度。
- 二维码四周保留安静区，不要贴边裁切。
- 使用圆角、圆点、线条或装饰定位点时，提高纠错等级。
- 不要把模块做得太细、太透明。
- 下载后用实际文件测试，而不仅仅看预览。

扫码器需要识别定位点、时序线、格式信息和足够的深色模块面积。只要这些结构被弱化，肉眼觉得好看也可能扫不出来。

## 什么时候适合使用带样式的二维码图片？

如果二维码会出现在公开营销物料上，比如海报、活动页、餐厅菜单、产品包装、优惠券、PPT 或社交图片，适当美化会让它更像品牌资产，而不是临时贴上去的工具图。

如果二维码用于仓储标签、票务闸机、发票、医疗说明或高频运营流程，则建议优先使用经典黑白二维码，尽量减少装饰。

## 推荐的二维码图片制作流程

1. 选择二维码类型。
2. 输入 URL、文本、Wi-Fi、图片链接或社交链接。
3. 选择清晰的视觉样式。
4. 保留足够安静区和高纠错等级。
5. 在最终尺寸下预览二维码图片。
6. 点击 Test this QR 测试是否可扫。
7. 日常使用下载 PNG，打印和设计使用 SVG。

## 常见问题

### 可以从图片生成二维码吗？

可以，但更准确地说，是为图片 URL 生成二维码。二维码不适合直接存储一张较大的原始图片。

### 可以从截图扫描二维码吗？

可以。只要截图足够清晰、二维码没有被裁切，浏览器端上传扫描就可以识别。

### SVG 是否比 PNG 更适合二维码？

用于打印或需要放大时，SVG 更合适；用于网页、文档和快速分享时，PNG 更方便。

### 为什么美化后的二维码图片扫不出来？

常见原因是对比度不足、安静区被裁掉、模块太小、压缩过重、定位点装饰过度，或模块形状让扫码器无法采样到足够的深色信号。
